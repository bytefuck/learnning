# 面试宝典生成器

## 概述

这是一个帮助用户生成结构化、系统性面试问答文档的skill。针对用户输入的技术点，由浅入深地生成面试问题和详细答案。

## 工作流程

### 1. 理解技术点

首先分析用户输入的技术点：
- 明确技术领域的范围
- 识别相关联的技术栈
- 确定技术点的核心概念

### 2. 创建目录结构

在用户当前工作目录下创建面试宝典目录：

```
面试宝典-{技术名称}/
├── README.md           # 汇总索引文件
├── 01-基础问题.md
├── 02-进阶问题.md
└── 03-实战问题.md
```

### 3. 内容生成规则

#### 基础问题章节 (01-基础问题.md)

包含5-8个问题，覆盖：
- 核心概念定义
- 基本工作原理
- 主要特性与用途
- 与相似技术的对比
- 基本使用方式

每个问题回答要求：
- 定义清晰准确
- 配合简单示例说明
- 篇幅200-400字

#### 进阶问题章节 (02-进阶问题.md)

包含5-8个问题，覆盖：
- 内部实现机制
- 性能优化原理
- 高级特性使用
- 架构设计考量
- 源码层面的理解

每个问题回答要求：
- 深入原理层面
- 配合架构图或流程图（使用Mermaid）
- 源码分析（如适用）
- 篇幅400-800字

#### 实战问题章节 (03-实战问题.md)

包含5-8个问题，覆盖：
- 实际项目中的应用场景
- 常见问题排查
- 性能调优实践
- 生产环境最佳实践
- 系统设计题目

每个问题回答要求：
- 结合真实场景
- 包含具体代码示例
- 问题诊断思路
- 篇幅600-1000字

### 4. 汇总索引文件 (README.md)

创建总览文件，包含：

```markdown
# {技术名称} 面试宝典

## 目录

- [基础问题](./01-基础问题.md)
- [进阶问题](./02-进阶问题.md)
- [实战问题](./03-实战问题.md)

## 概述

{技术简介，100-200字}

## 知识图谱

{使用Mermaid绘制技术点的关系图}

## 学习建议

{针对不同水平的读者给出学习路径建议}
```

## 问题格式模板

每个问题使用以下格式：

```markdown
## Q{n}: {问题标题}

**问题**：{完整问题}

**答案**：

{详细解答}

{如需代码示例}

\`\`\`{语言}
// 代码示例
\`\`\`

{如需图表}

\`\`\`mermaid
graph TD
    A[概念] --> B[实现]
\`\`\`

**要点总结**：
- {要点1}
- {要点2}
- {要点3}

---
```

## 质量标准

1. **准确性**：所有技术内容必须准确无误
2. **深度**：进阶和实战问题要有足够深度
3. **实用性**：答案要能帮助读者真正理解
4. **结构化**：层次分明，便于阅读
5. **完整性**：覆盖该技术点的主要方面

## 执行步骤

1. 确认用户要学习的技术点
2. 创建面试宝典目录
3. 按顺序生成三个章节文件
4. 创建汇总索引文件
5. 告知用户文件位置和结构

## 注意事项

- 如果技术点范围过大（如"Java"），建议用户聚焦到具体模块
- 可以根据技术点的复杂度调整问题数量
- 对于快速发展的技术，注意内容时效性
- 适当参考官方文档和权威资料
