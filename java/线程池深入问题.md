# 线程池深入问题

## 1. ThreadPoolExecutor 中的 Worker 类是如何设计的？为什么继承 AQS？

### Worker 类的结构

```java
private final class Worker
        extends AbstractQueuedSynchronizer
        implements Runnable {
    
    // 工作线程
    final Thread thread;
    
    // 初始任务（可能为 null）
    Runnable firstTask;
    
    // 完成的任务计数
    volatile long completedTasks;

    Worker(Runnable firstTask) {
        setState(-1);  // 禁止中断，直到 runWorker
        this.firstTask = firstTask;
        this.thread = getThreadFactory().newThread(this);
    }

    public void run() {
        runWorker(this);
    }

    // 锁相关方法
    protected boolean isHeldExclusively() {
        return getState() != 0;
    }

    protected boolean tryAcquire(int unused) {
        if (compareAndSetState(0, 1)) {
            setExclusiveOwnerThread(Thread.currentThread());
            return true;
        }
        return false;
    }

    protected boolean tryRelease(int unused) {
        setExclusiveOwnerThread(null);
        setState(0);
        return true;
    }

    void lock()        { acquire(1); }
    boolean tryLock()  { return tryAcquire(1); }
    void unlock()      { release(1); }
    boolean isLocked() { return isHeldExclusively(); }
}
```

### Worker 继承 AQS 的原因

**1. 实现独占锁控制**
- Worker 使用 AQS 实现了一个简单的独占锁
- lock()/unlock() 控制任务的执行状态

**2. 防止任务执行时被中断**
```java
final void runWorker(Worker w) {
    Thread wt = Thread.currentThread();
    Runnable task = w.firstTask;
    w.firstTask = null;
    w.unlock();  // 允许中断
    boolean completedAbruptly = true;
    try {
        while (task != null || (task = getTask()) != null) {
            w.lock();  // 加锁，防止中断
            // ... 执行任务
            w.unlock();
        }
        completedAbruptly = false;
    } finally {
        processWorkerExit(w, completedAbruptly);
    }
}
```

**3. 追踪任务执行状态**
- 通过锁状态判断 Worker 是否在执行任务
- isLocked() 检查 Worker 是否空闲

### Worker 的状态管理

```
Worker 状态转换：

创建时：state = -1（禁止中断）
   |
   v
runWorker 调用 unlock()：state = 0（允许中断）
   |
   v
执行任务前 lock()：state = 1（正在执行）
   |
   v
执行任务后 unlock()：state = 0（空闲）
   |
   v
获取新任务或退出
```

### Worker 与线程池的关系

```
ThreadPoolExecutor
        |
        +-- workers: HashSet<Worker>  （所有工作线程）
        |
        +-- workQueue: BlockingQueue  （任务队列）
        |
        v
    Worker (继承 AQS)
        |
        +-- thread: Thread      （实际线程）
        |
        +-- firstTask: Runnable （初始任务）
        |
        +-- completedTasks      （完成任务数）
```

## 2. 线程池如何保证线程安全地添加任务和创建线程？

### 核心状态变量 ctl

```java
// ctl 是一个原子变量，包含两部分信息：
// 1. 高3位：线程池运行状态 (runState)
// 2. 低29位：工作线程数量 (workerCount)

private final AtomicInteger ctl = new AtomicInteger(ctlOf(RUNNING, 0));

// 线程数位数
private static final int COUNT_BITS = Integer.SIZE - 3;  // 29

// 线程数最大值
private static final int CAPACITY   = (1 << COUNT_BITS) - 1;

// 线程池状态
private static final int RUNNING    = -1 << COUNT_BITS;  // 111...000
private static final int SHUTDOWN   =  0 << COUNT_BITS;  // 000...000
private static final int STOP       =  1 << COUNT_BITS;  // 001...000
private static final int TIDYING    =  2 << COUNT_BITS;  // 010...000
private static final int TERMINATED =  3 << COUNT_BITS;  // 011...000
```

### 状态计算方法

```java
// 获取运行状态
private static int runStateOf(int c)     { return c & ~CAPACITY; }

// 获取线程数
private static int workerCountOf(int c)  { return c & CAPACITY; }

// 计算 ctl 值
private static int ctlOf(int rs, int wc) { return rs | wc; }
```

### 线程安全的实现方式

**1. CAS 操作保证原子性**
```java
// 增加线程数
private boolean compareAndIncrementWorkerCount(int expect) {
    return ctl.compareAndSet(expect, expect + 1);
}

// 减少线程数
private boolean compareAndDecrementWorkerCount(int expect) {
    return ctl.compareAndSet(expect, expect - 1);
}
```

**2. ReentrantLock 保护关键操作**
```java
private final ReentrantLock mainLock = new ReentrantLock();

// 添加 Worker
private boolean addWorker(Runnable firstTask, boolean core) {
    // ... CAS 操作
    
    mainLock.lock();
    try {
        // 保护 workers 集合的修改
        workers.add(w);
        // ... 更新统计信息
    } finally {
        mainLock.unlock();
    }
}
```

**3. 任务队列使用线程安全的 BlockingQueue**
```java
// 常用实现
LinkedBlockingQueue    // 链表阻塞队列
ArrayBlockingQueue     // 数组阻塞队列
SynchronousQueue       // 同步队列
PriorityBlockingQueue  // 优先级阻塞队列
```

### addWorker 的线程安全实现

```java
private boolean addWorker(Runnable firstTask, boolean core) {
    retry:
    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

        // 检查线程池状态
        if (rs >= SHUTDOWN &&
            !(rs == SHUTDOWN && firstTask == null && !workQueue.isEmpty()))
            return false;

        for (;;) {
            int wc = workerCountOf(c);
            // 检查线程数限制
            if (wc >= CAPACITY ||
                wc >= (core ? corePoolSize : maximumPoolSize))
                return false;
            // CAS 增加线程数（保证线程安全）
            if (compareAndIncrementWorkerCount(c))
                break retry;
            // 重新检查状态
            c = ctl.get();
            if (runStateOf(c) != rs)
                continue retry;
        }
    }

    boolean workerStarted = false;
    boolean workerAdded = false;
    Worker w = null;
    try {
        w = new Worker(firstTask);
        final Thread t = w.thread;
        if (t != null) {
            final ReentrantLock mainLock = this.mainLock;
            mainLock.lock();  // 加锁保护
            try {
                int rs = runStateOf(ctl.get());
                if (rs < SHUTDOWN ||
                    (rs == SHUTDOWN && firstTask == null)) {
                    if (t.isAlive())
                        throw new IllegalThreadStateException();
                    workers.add(w);  // 线程安全地添加
                    int s = workers.size();
                    if (s > largestPoolSize)
                        largestPoolSize = s;
                    workerAdded = true;
                }
            } finally {
                mainLock.unlock();
            }
            if (workerAdded) {
                t.start();
                workerStarted = true;
            }
        }
    } finally {
        if (!workerStarted)
            addWorkerFailed(w);
    }
    return workerStarted;
}
```

### execute 方法的线程安全

```java
public void execute(Runnable command) {
    if (command == null)
        throw new NullPointerException();
    
    int c = ctl.get();
    
    // 第一步：CAS 竞争创建核心线程
    if (workerCountOf(c) < corePoolSize) {
        if (addWorker(command, true))
            return;
        c = ctl.get();  // 重新获取状态
    }
    
    // 第二步：线程安全地加入队列
    if (isRunning(c) && workQueue.offer(command)) {
        int recheck = ctl.get();
        // 双重检查
        if (!isRunning(recheck) && remove(command))
            reject(command);
        else if (workerCountOf(recheck) == 0)
            addWorker(null, false);
    }
    // 第三步：尝试创建非核心线程
    else if (!addWorker(command, false))
        reject(command);
}
```

## 3. shutdown() 和 shutdownNow() 的区别是什么？

### 状态转换图

```
RUNNING -> SHUTDOWN -> TIDYING -> TERMINATED
    |         |
    v         v
  STOP    -> TIDYING -> TERMINATED
```

### shutdown() 方法

```java
public void shutdown() {
    final ReentrantLock mainLock = this.mainLock;
    mainLock.lock();
    try {
        checkShutdownAccess();
        // 将状态设为 SHUTDOWN
        advanceRunState(SHUTDOWN);
        // 中断空闲线程
        interruptIdleWorkers();
        // 钩子方法
        onShutdown();
    } finally {
        mainLock.unlock();
    }
    tryTerminate();
}
```

**特点**：
1. 状态变为 SHUTDOWN
2. 不再接受新任务
3. 执行完已提交的任务和队列中的任务
4. 中断空闲线程

### shutdownNow() 方法

```java
public List<Runnable> shutdownNow() {
    List<Runnable> tasks;
    final ReentrantLock mainLock = this.mainLock;
    mainLock.lock();
    try {
        checkShutdownAccess();
        // 将状态设为 STOP
        advanceRunState(STOP);
        // 中断所有线程
        interruptWorkers();
        // 返回队列中的任务
        tasks = drainQueue();
    } finally {
        mainLock.unlock();
    }
    tryTerminate();
    return tasks;
}
```

**特点**：
1. 状态变为 STOP
2. 不再接受新任务
3. 尝试中断正在执行的任务
4. 返回队列中等待的任务

### 详细对比

| 特性 | shutdown() | shutdownNow() |
|------|------------|---------------|
| 状态变化 | RUNNING -> SHUTDOWN | RUNNING -> STOP |
| 接受新任务 | 否 | 否 |
| 执行队列任务 | 是 | 否 |
| 中断正在执行的任务 | 否 | 是 |
| 中断空闲线程 | 是 | 是 |
| 返回值 | void | List<Runnable> |

### 中断策略

**shutdown() 中断空闲线程**：
```java
private void interruptIdleWorkers() {
    interruptIdleWorkers(false);
}

private void interruptIdleWorkers(boolean onlyOne) {
    final ReentrantLock mainLock = this.mainLock;
    mainLock.lock();
    try {
        for (Worker w : workers) {
            Thread t = w.thread;
            // 只中断未加锁的线程（空闲线程）
            if (!t.isInterrupted() && w.tryLock()) {
                try {
                    t.interrupt();
                } catch (SecurityException ignore) {
                } finally {
                    w.unlock();
                }
            }
            if (onlyOne)
                break;
        }
    } finally {
        mainLock.unlock();
    }
}
```

**shutdownNow() 中断所有线程**：
```java
private void interruptWorkers() {
    final ReentrantLock mainLock = this.mainLock;
    mainLock.lock();
    try {
        for (Worker w : workers) {
            Thread t = w.thread;
            // 中断所有线程
            if (!t.isInterrupted() && t != null)
                t.interrupt();
        }
    } finally {
        mainLock.unlock();
    }
}
```

### 使用示例

```java
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100)
);

// 优雅关闭
executor.shutdown();
try {
    // 等待任务完成，最多等待 60 秒
    if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
        // 超时后强制关闭
        executor.shutdownNow();
    }
} catch (InterruptedException e) {
    executor.shutdownNow();
    Thread.currentThread().interrupt();
}
```

### 线程池状态详解

| 状态 | 值 | 说明 |
|------|-----|------|
| RUNNING | -1 << 29 | 接受新任务，处理队列任务 |
| SHUTDOWN | 0 << 29 | 不接受新任务，处理队列任务 |
| STOP | 1 << 29 | 不接受新任务，不处理队列任务，中断正在执行的任务 |
| TIDYING | 2 << 29 | 所有任务终止，workerCount 为 0，执行 terminated() |
| TERMINATED | 3 << 29 | terminated() 执行完成 |

### terminated() 钩子方法

```java
protected void terminated() { }

// 状态转换到 TIDYING
final void tryTerminate() {
    for (;;) {
        int c = ctl.get();
        if (isRunning(c) ||
            runStateAtLeast(c, TIDYING) ||
            (runStateOf(c) == SHUTDOWN && !workQueue.isEmpty()))
            return;
        if (workerCountOf(c) != 0) {
            interruptIdleWorkers(ONLY_ONE);
            return;
        }
        final ReentrantLock mainLock = this.mainLock;
        mainLock.lock();
        try {
            if (ctl.compareAndSet(c, ctlOf(TIDYING, 0))) {
                try {
                    terminated();  // 执行钩子方法
                } finally {
                    ctl.set(ctlOf(TERMINATED, 0));
                    termination.signalAll();
                }
                return;
            }
        } finally {
            mainLock.unlock();
        }
    }
}
