# 线程池进阶问题

## 1. 线程池的工作流程是怎样的？（任务提交后的处理流程）

### 整体流程图

```
                        提交任务 (execute/submit)
                                 |
                                 v
                    +---------------------------+
                    | 当前线程数 < corePoolSize? |
                    +---------------------------+
                          /            \
                        是              否
                        /                \
                       v                  v
               创建核心线程         +-------------------+
                                   | 队列是否已满?      |
                                   +-------------------+
                                     /          \
                                   否            是
                                   /              \
                                  v                v
                             加入队列      +---------------------------+
                                          | 当前线程数 < maximumPoolSize? |
                                          +---------------------------+
                                              /             \
                                            是               否
                                            /                 \
                                           v                   v
                                   创建非核心线程        执行拒绝策略
```

### 详细流程说明

**Step 1: 检查核心线程数**
```java
public void execute(Runnable command) {
    if (command == null)
        throw new NullPointerException();
        
    int c = ctl.get();
    
    // 1. 当前线程数 < 核心线程数，直接创建核心线程
    if (workerCountOf(c) < corePoolSize) {
        if (addWorker(command, true))
            return;
        c = ctl.get();
    }
    
    // ... 后续步骤
}
```

**Step 2: 加入工作队列**
```java
public void execute(Runnable command) {
    // ... Step 1
    
    // 2. 线程池运行中，将任务加入队列
    if (isRunning(c) && workQueue.offer(command)) {
        int recheck = ctl.get();
        
        // 二次检查：如果线程池已关闭，移除任务并执行拒绝策略
        if (!isRunning(recheck) && remove(command))
            reject(command);
        // 如果没有工作线程，创建一个
        else if (workerCountOf(recheck) == 0)
            addWorker(null, false);
    }
    // ... Step 3
}
```

**Step 3: 创建非核心线程或拒绝**
```java
public void execute(Runnable command) {
    // ... Step 1, 2
    
    // 3. 队列已满，尝试创建非核心线程
    else if (!addWorker(command, false))
        // 4. 创建失败（达到最大线程数），执行拒绝策略
        reject(command);
}
```

### 核心源码分析

**execute 方法完整版**：
```java
public void execute(Runnable command) {
    if (command == null)
        throw new NullPointerException();
    
    int c = ctl.get();
    
    // 情况1：线程数 < 核心线程数，创建核心线程
    if (workerCountOf(c) < corePoolSize) {
        if (addWorker(command, true))
            return;
        c = ctl.get();
    }
    
    // 情况2：线程数 >= 核心线程数，加入队列
    if (isRunning(c) && workQueue.offer(command)) {
        int recheck = ctl.get();
        if (!isRunning(recheck) && remove(command))
            reject(command);
        else if (workerCountOf(recheck) == 0)
            addWorker(null, false);
    }
    // 情况3：队列满，创建非核心线程
    else if (!addWorker(command, false))
        // 情况4：创建失败，执行拒绝策略
        reject(command);
}
```

**addWorker 方法**：
```java
private boolean addWorker(Runnable firstTask, boolean core) {
    retry:
    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

        // 检查线程池状态
        if (rs >= SHUTDOWN &&
            !(rs == SHUTDOWN && firstTask == null && !workQueue.isEmpty()))
            return false;

        for (;;) {
            int wc = workerCountOf(c);
            // 检查线程数限制
            if (wc >= CAPACITY ||
                wc >= (core ? corePoolSize : maximumPoolSize))
                return false;
            // CAS 增加线程数
            if (compareAndIncrementWorkerCount(c))
                break retry;
            c = ctl.get();
            if (runStateOf(c) != rs)
                continue retry;
        }
    }

    boolean workerStarted = false;
    boolean workerAdded = false;
    Worker w = null;
    try {
        w = new Worker(firstTask);
        final Thread t = w.thread;
        if (t != null) {
            final ReentrantLock mainLock = this.mainLock;
            mainLock.lock();
            try {
                int rs = runStateOf(ctl.get());
                if (rs < SHUTDOWN ||
                    (rs == SHUTDOWN && firstTask == null)) {
                    if (t.isAlive())
                        throw new IllegalThreadStateException();
                    workers.add(w);
                    int s = workers.size();
                    if (s > largestPoolSize)
                        largestPoolSize = s;
                    workerAdded = true;
                }
            } finally {
                mainLock.unlock();
            }
            if (workerAdded) {
                t.start();
                workerStarted = true;
            }
        }
    } finally {
        if (!workerStarted)
            addWorkerFailed(w);
    }
    return workerStarted;
}
```

## 2. corePoolSize 和 maximumPoolSize 什么时候生效？

### 生效时机

| 参数 | 生效时机 | 说明 |
|------|----------|------|
| corePoolSize | 任务提交时，线程数 < corePoolSize | 直接创建核心线程 |
| maximumPoolSize | 队列满时，线程数 < maximumPoolSize | 创建非核心线程 |

### 详细说明

**corePoolSize 生效场景**：
1. 提交任务时，当前线程数 < corePoolSize
2. 直接创建新的核心线程执行任务
3. 即使有空闲线程，也会创建新线程

**maximumPoolSize 生效场景**：
1. 提交任务时，线程数 >= corePoolSize
2. 且任务队列已满
3. 当前线程数 < maximumPoolSize
4. 创建非核心线程执行任务

### 图解说明

```
场景1：线程数 < corePoolSize
------------------------------
提交任务 -> 直接创建核心线程执行

场景2：corePoolSize <= 线程数 < maximumPoolSize 且队列未满
----------------------------------------------------------
提交任务 -> 加入队列等待

场景3：corePoolSize <= 线程数 < maximumPoolSize 且队列已满
----------------------------------------------------------
提交任务 -> 创建非核心线程执行

场景4：线程数 >= maximumPoolSize 且队列已满
--------------------------------------------
提交任务 -> 执行拒绝策略
```

### 特殊情况

**allowCoreThreadTimeOut = true**：
```java
// 允许核心线程超时回收
executor.allowCoreThreadTimeOut(true);
```
- 核心线程也会在空闲时间超过 keepAliveTime 后被回收
- 线程数可能降至 0

**prestartAllCoreThreads**：
```java
// 预启动所有核心线程
executor.prestartAllCoreThreads();
```
- 提前创建所有核心线程
- 即使没有任务也会创建

## 3. 线程池中线程是如何被回收的？keepAliveTime 参数的作用？

### 线程回收机制

线程池中的线程回收主要通过 **Worker** 类的 **getTask** 方法实现：

```java
private Runnable getTask() {
    boolean timedOut = false;

    for (;;) {
        int c = ctl.get();
        int rs = runStateOf(c);

        // 检查线程池状态
        if (rs >= SHUTDOWN && (rs >= STOP || workQueue.isEmpty())) {
            decrementWorkerCount();
            return null;
        }

        int wc = workerCountOf(c);

        // 是否允许超时回收
        boolean timed = allowCoreThreadTimeOut || wc > corePoolSize;

        // 回收条件：
        // 1. 线程数超过最大值 或 允许超时且已超时
        // 2. 工作队列为空 或 只有一个线程
        if ((wc > maximumPoolSize || (timed && timedOut))
            && (wc > 1 || workQueue.isEmpty())) {
            if (compareAndDecrementWorkerCount(c))
                return null;
            continue;
        }

        try {
            // 从队列获取任务
            Runnable r = timed ?
                workQueue.poll(keepAliveTime, TimeUnit.NANOSECONDS) :  // 超时获取
                workQueue.take();  // 阻塞获取
            if (r != null)
                return r;
            timedOut = true;
        } catch (InterruptedException retry) {
            timedOut = false;
        }
    }
}
```

### 回收条件

**非核心线程回收**：
1. 当前线程数 > corePoolSize
2. 空闲时间超过 keepAliveTime
3. workQueue.poll() 返回 null

**核心线程回收**（需开启）：
1. allowCoreThreadTimeOut = true
2. 空闲时间超过 keepAliveTime
3. workQueue.poll() 返回 null

### keepAliveTime 的作用

```
keepAliveTime：非核心线程空闲时的存活时间

            提交任务
               |
               v
    +----------------------+
    | 创建非核心线程        |
    +----------------------+
               |
               v
    +----------------------+
    | 执行任务              |
    +----------------------+
               |
               v
    +----------------------+
    | 从队列获取任务        |
    +----------------------+
               |
        +------+------+
        |             |
    有任务          无任务
        |             |
        v             v
    继续执行    poll(keepAliveTime)
                        |
                 +------+------+
                 |             |
             获取到任务     超时未获取
                 |             |
                 v             v
             继续执行       线程回收
```

### Worker 运行流程

```java
final void runWorker(Worker w) {
    Thread wt = Thread.currentThread();
    Runnable task = w.firstTask;
    w.firstTask = null;
    w.unlock();  // 允许中断
    boolean completedAbruptly = true;
    try {
        // 循环获取任务
        while (task != null || (task = getTask()) != null) {
            w.lock();
            // 检查中断
            if ((runStateAtLeast(ctl.get(), STOP) ||
                 (Thread.interrupted() &&
                  runStateAtLeast(ctl.get(), STOP))) &&
                !wt.isInterrupted())
                wt.interrupt();
            try {
                beforeExecute(wt, task);
                Throwable thrown = null;
                try {
                    task.run();  // 执行任务
                } catch (RuntimeException x) {
                    thrown = x; throw x;
                } catch (Error x) {
                    thrown = x; throw x;
                } catch (Throwable x) {
                    thrown = x; throw new Error(x);
                } finally {
                    afterExecute(task, thrown);
                }
            } finally {
                task = null;
                w.completedTasks++;
                w.unlock();
            }
        }
        completedAbruptly = false;
    } finally {
        // 线程退出处理
        processWorkerExit(w, completedAbruptly);
    }
}
```

### 线程回收时机总结

| 条件 | 是否回收 | 说明 |
|------|----------|------|
| 线程数 > corePoolSize 且空闲超时 | 是 | 非核心线程默认回收 |
| 线程数 <= corePoolSize 且 allowCoreThreadTimeOut=true 且空闲超时 | 是 | 核心线程也可回收 |
| 线程数 <= corePoolSize 且 allowCoreThreadTimeOut=false | 否 | 核心线程不回收 |
| 线程执行异常 | 是 | completedAbruptly=true |
| 线程池关闭 | 是 | 所有线程回收 |

### 配置建议

```java
// 默认配置：核心线程不回收
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    5, 10, 60L, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(100)
);

// 允许核心线程回收
executor.allowCoreThreadTimeOut(true);
// 此时 keepAliveTime 对核心线程也生效

// 设置最小线程数（Java 8+）
executor.setMinimumPoolSize(2);
```
